# Kocom Wallpad RS485 테스트 도구

`kocom-wallpad-rs485` 애드온 개발 및 디버깅을 위한 독립 실행형 Python 스크립트 모음입니다.  
별도의 Home Assistant 환경 없이 PC에서 직접 RS485 버스를 분석할 수 있습니다.

---

## 파일 목록

| 파일 | 설명 |
|------|------|
| `packet-log.py` | RS485 시리얼/소켓으로 들어오는 모든 패킷을 실시간 캡처하여 16진수로 터미널에 출력 |
| `translate-packet.py` | 16진수 패킷을 stdin으로 입력받아 사람이 알아보기 쉬운 형태로 해석하여 출력 |

---

## 패킷 구조

코콤 월패드 RS485 패킷은 항상 **21바이트** 고정 길이입니다.

```
Byte  0- 1 : AA 55          (프리픽스, 패킷 시작)
Byte  2    : 30             (고정)
Byte  3    : BC / DC        (상위 니블 = 패킷타입: 0x0B=명령/조회, 0x0D=응답/ACK)
Byte  4    : 00             (고정)
Byte  5    : dest 장치 코드
Byte  6    : dest 방 코드
Byte  7    : src  장치 코드
Byte  8    : src  방 코드
Byte  9    : 커맨드 바이트
Byte 10-17 : 페이로드 (8바이트)
Byte 18    : 체크섬 = sum(Byte 2~17) % 256
Byte 19-20 : 0D 0D          (서픽스, 패킷 종료)
```

### 장치 코드

| 코드 | 장치 |
|------|------|
| `0x01` | 월패드 |
| `0x0E` | 조명 |
| `0x3B` | 콘센트 |
| `0x36` | 온도조절기 |
| `0x39` | 에어컨 |
| `0x48` | 환기장치 |
| `0x2C` | 가스밸브 |
| `0x44` | 엘리베이터 |
| `0x60` | 동작감지기 |
| `0x98` | 공기질센서 |

### 방 코드

| 코드 | 방 |
|------|-----|
| `0x00` | 거실 |
| `0x01` | 방1 |
| `0x02` | 방2 |
| `0x03` | 방3 |
| `0x04` | 주방 |
| `0xFF` | 전체 |

## packet-log.py

RS485 시리얼 포트 또는 TCP 소켓에 연결하여 수신되는 모든 패킷을 실시간으로 캡처하고 터미널에 16진수로 출력합니다.

### 의존성 설치

```bash
# 소켓 모드만 사용하는 경우 추가 패키지 불필요

# 시리얼 모드를 사용하는 경우
pip install pyserial-asyncio
```

### 옵션

```
--type        연결 방식: socket (기본) 또는 serial
--host        소켓 서버 IP 주소 (기본: 192.168.1.100)
--port        소켓 포트 번호 (기본: 8899)
--serial-port 시리얼 포트 경로 (기본: /dev/ttyUSB0)
--baud        시리얼 보드레이트 (기본: 9600)
```

### 사용 방법

**소켓 모드 (EW11 등 RS485-to-WiFi 장치)**

```bash
python packet-log.py --type socket --host 192.168.1.100 --port 8899
```

**시리얼 모드 (USB-RS485 동글)**

```bash
python packet-log.py --type serial --serial-port /dev/ttyUSB0
python packet-log.py --type serial --serial-port /dev/ttyUSB0 --baud 9600
```

**Windows 시리얼**

```bash
python packet-log.py --type serial --serial-port COM3
```

### 출력 예시

```
============================================================
  Kocom Wallpad RS485 패킷 로거
  패킷 형식: AA 55 ... 0D 0D  (21B)
  종료: Ctrl+C
============================================================
[소켓] 192.168.1.100:8899 연결 중...
[소켓] 연결됨 — 패킷 캡처 시작
[14:23:01.452] #0001  AA 55 30 BC 00 0E 00 01 00 3A 00 00 00 00 00 00 00 00 35 0D 0D
[14:23:01.530] #0002  AA 55 30 BC 00 01 00 0E 00 00 FF 00 00 00 00 00 00 00 FA 0D 0D
[14:23:02.105] #0003  AA 55 30 BC 00 36 01 01 00 00 11 00 16 00 14 00 00 00 5F 0D 0D
```

각 패킷은 `[시각] #번호  16진수` 형태로 출력됩니다. 패킷 사이의 잡음 바이트는 `[noise Nb]`로 별도 표시됩니다.

연결이 끊기면 자동으로 재연결을 시도합니다. 재연결 대기 시간은 최초 5초에서 최대 60초까지 지수적으로 증가합니다.

## translate-packet.py

stdin으로 16진수 패킷을 입력받아 필드를 분해하고 장치별 페이로드를 해석하여 출력합니다.

### 의존성 설치

표준 라이브러리만 사용하므로 추가 패키지가 필요 없습니다.

### 입력 형식

대소문자를 구분하지 않으며 다음 세 가지 형식을 모두 지원합니다.

```
AA 55 30 BC 00 0E 00 01 00 3A 00 00 00 00 00 00 00 00 35 0D 0D   (공백 구분)
AA5530BC000E0001003A0000000000000000350D0D                        (연속)
AA-55-30-BC-00-0E-00-01-00-3A-00-00-00-00-00-00-00-00-35-0D-0D   (하이픈 구분)
```

- `#`으로 시작하는 줄은 주석으로 처리됩니다.
- 21바이트보다 긴 입력은 버스 스트림으로 간주하여 `AA 55` 기준으로 패킷을 자동 분리합니다.

### 사용 방법

**직접 입력 (대화형)**

```bash
python translate-packet.py
# 프롬프트에 패킷 16진수를 입력하고 Enter
```

**파이프 (echo)**

```bash
echo "AA 55 30 BC 00 0E 00 01 00 3A 00 00 00 00 00 00 00 00 35 0D 0D" | python translate-packet.py
```

**파일로부터 일괄 처리**

```bash
# packets.txt에 패킷을 한 줄씩 저장한 뒤
cat packets.txt | python translate-packet.py
```

**packet-log.py와 연동 (실시간 해석)**

```bash
python packet-log.py --type socket --host 192.168.1.100 --port 8899 | python translate-packet.py
```

> `packet-log.py` 출력의 `#0001  AA 55 ...` 형식에서 `AA 55`보다 앞의 텍스트는 무시되고 패킷만 자동으로 추출됩니다.

### 출력 예시

**거실 조명 상태 조회 (월패드 → 조명)**

```
╔══════════════════════════════════════════════════════════════════╗
║  패킷 : AA 55 30 BC 00 0E 00 01 00 3A 00 00 00 00 00 00 00 00 35 0D 0D
╠══════════════════════════════════════════════════════════════════╣
║  상태 : ✓ 유효
╠══════════════════════════════════════════════════════════════════╣
║  패킷타입 :  0x0B  명령/조회
║  수신지   :  조명(0x0E)  방=거실(0x00)
║  발신지   :  월패드(0x01)  방=거실(0x00)
║  방향     :  월패드 → 조명(0x0E) (거실(0x00))
║  ──────────────────────────────────────────────────────────────────
║  커맨드   :  0x3A  조회
║  페이로드 :  00 00 00 00 00 00 00 00
║  체크섬   :  0x35  (계산값: 0x35)  ✓
╠══════════════════════════════════════════════════════════════════╣
║  조명 (거실) 상태 조회
╚══════════════════════════════════════════════════════════════════╝
```

**거실 조명 상태 응답 (조명 → 월패드, #1 켜짐)**

```

║  조명 (거실)  [장치 상태 응답]
║    켜진 채널: #1
║    채널 상태: #1 ON  | #2 off | #3 off | #4 off | #5 off | #6 off | #7 off | #8 off
╚══════════════════════════════════════════════════════════════════╝
```

**방1 온도조절기 (난방 중, 현재 20°C, 설정 22°C)**

```
╠══════════════════════════════════════════════════════════════════╣
║  온도조절기 #1: 모드=난방  상태=일반
║    현재온도: 20.0°C  |  설정온도: 22.0°C
╚══════════════════════════════════════════════════════════════════╝
```

**환기장치 (auto 모드, 중풍, CO2 882ppm)**

```
╠══════════════════════════════════════════════════════════════════╣
║  환기장치 (거실): 운전 중
║    모드: auto(자동)  |  풍량: 중풍(medium)
║    CO2: 882 ppm
╚══════════════════════════════════════════════════════════════════╝
```

### 해석 지원 장치

| 장치 | 해석 내용 |
|------|-----------|
| 조명 | 채널별 ON/OFF 상태 (최대 8채널) |
| 콘센트 | 채널별 ON/OFF 상태 (최대 8채널) |
| 조명 차단기 | 전체 차단 ON/OFF |
| 온도조절기 | 난방 모드, 외출 여부, 현재·설정·온수·난방 온도, 에러코드 |
| 에어컨 | 운전 여부, 운전 모드(냉방/송풍/제습/자동), 팬 속도, 현재·설정 온도 |
| 환기장치 | 운전 여부, 프리셋 모드, 풍량, 타이머, CO2 농도, 에러코드 |
| 가스밸브 | 열림/잠김 상태 |
| 엘리베이터 | 방향(대기/하강/상승/도착), 현재 층수 |
| 동작감지기 | 동작 감지 여부 |
| 공기질센서 | PM10, PM2.5, CO2, VOC, 온도, 습도 |

## 실전 활용 예시

### 패킷 캡처 후 파일로 저장, 나중에 분석

```bash
# 캡처 (타임스탬프 포함하여 저장)
python packet-log.py --type socket --host 192.168.1.100 | tee packets.log

# 저장된 로그에서 패킷만 추출하여 해석
grep -oP '(?<=#\d{4}  )[A-F0-9 ]+' packets.log | python translate-packet.py
```

### 특정 장치 패킷만 필터링

```bash
# 온도조절기(0x36) 패킷만 보기
python packet-log.py --type socket --host 192.168.1.100 | grep "36"
```

### 체크섬 계산 검증

```python
raw = bytes.fromhex("AA5530BC000E0001003A0000000000000000350D0D")
chk = sum(raw[2:18]) % 256
print(f"체크섬: 0x{chk:02X}")  # → 0x35
```
